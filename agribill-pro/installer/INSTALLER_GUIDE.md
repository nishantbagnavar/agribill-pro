# AgriBill Pro — Windows Installer Build Guide

## Prerequisites

1. **Inno Setup 6.x** — https://jrsoftware.org/isinfo.php
2. **Node.js 20 standalone binary** — download from https://nodejs.org/dist/v20.x.x/node-v20.x.x-win-x64.zip
3. **nssm** (Windows service manager) — https://nssm.cc/download

## Directory layout expected before building

```
installer/
├── assets/
│   ├── node/               ← extract node-v20 standalone here (contains node.exe)
│   ├── nssm/               ← place nssm.exe here
│   ├── icon.ico            ← 256x256 app icon
│   ├── wizard-small.bmp    ← 55x58 Inno Setup wizard image
│   ├── wizard-large.bmp    ← 164x314 Inno Setup wizard side image
│   ├── agribill-launcher.bat
│   ├── install-service.bat
│   └── uninstall-service.bat
├── output/                 ← generated by iscc (auto-created)
├── agribill-pro.iss
└── INSTALLER_GUIDE.md
```

## Build steps

```bash
# 1. Build the React frontend
cd agribill-pro
npm run build

# 2. Install backend production deps
cd backend
npm install --omit=dev
cd ..

# 3. Compile the installer (from project root)
iscc installer\agribill-pro.iss
```

The output will be at `installer\output\AgriBillPro-Setup-1.0.0.exe`.

## What the installer does

1. Copies app files to `C:\Program Files\AgriBillPro\`
2. Keeps user data in `%LOCALAPPDATA%\AgriBillPro\data\` (survives upgrades)
3. Registers **AgriBillPro** as a Windows service via nssm (auto-start)
4. Runs DB migrations on first install
5. Creates Start Menu + optional desktop shortcut
6. Opens http://localhost:5000 in the browser after install

## Updating to a new version

### Full update (replace everything)
1. Bump `version` in root `package.json`
2. Re-run build steps above
3. Run `iscc installer\agribill-pro.iss` — Inno Setup detects existing install and upgrades

### Delta update (ship only changed files)
```bash
node scripts/create-delta.js \
  --from ./releases/1.0.0 \
  --to   ./releases/1.1.0 \
  --out  delta-1.0.0-to-1.1.0.zip
```
Upload the zip to your CDN, then update `update-manifest.json`:
- Bump `latestVersion`
- Add entry to `deltaZips` array with `fromVersion`, `toVersion`, `url`, `size`, `sha256`

## Service management (manual)

```bash
# Start
net start AgriBillPro
# Stop
net stop AgriBillPro
# View logs
type %LOCALAPPDATA%\AgriBillPro\logs\app.log
```
